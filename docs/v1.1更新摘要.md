## 📊 对比：提取 vs 重构

### ❌ 简单提取（机械，不推荐）

```markdown
用户说："这个方案不行"
领导说："我觉得可以试试"
```

**问题**:
- 像流水账，毫无洞察
- 缺少场景感和上下文
- 看不出关系动态

---

### ✅ 场景重构（智能，推荐）

```markdown
会议进入方案选择环节。当项目经理提出A方案时，技术负责人
立即表达了明确的顾虑，认为技术实现有风险。

此时领导者介入讨论，用一句"我觉得可以试试"定下了基调——
鼓励创新尝试。技术负责人在领导表态后没有继续坚持反对，
显示出对层级决策的尊重。

> 事实依据：
> - "这个方案不行"（技术负责人）
> - "我觉得可以试试"（领导）
> - 会议记录显示此后无人继续反对
```

**优势**:
- ✅ 还原了对话的张力和转折
- ✅ 展现了关系动态和决策逻辑
- ✅ 既客观（基于事实）又有洞察（不机械）

---

## 🎯 三大原则

### 1. 忠于事实
- 所有重构必须有原文依据
- 不添加原文中不存在的细节
- 标注事实依据

### 2. 还原场景
- 重现对话的张力和转折
- 呈现关系动态
- 展示决策演进

### 3. 保持客观
- 不做价值判断
- 不臆测内心想法
- 不编造动机

---

## 📐 重构的四个维度

```
1. 对话张力
   └─ 分歧、转折、共识

2. 关系动态
   └─ 权力、立场、情感

3. 情感氛围
   └─ 紧张、协作、急迫

4. 决策逻辑
   └─ 问题 → 讨论 → 决策
```

## 🛡️ 防止"过度重构"的边界

### 可追溯性测试

> **核心原则**: 重构的每一个描述，都应该能追溯到原文的某个细节

| 可以做 ✅ | 不可以做 ❌ |
|---------|-----------|
| 还原对话的张力和转折 | 添加原文中不存在的情节 |
| 描述可观察的关系动态 | 臆测参与者的内心想法 |
| 基于用词推断情感氛围 | 编造参与者的动机和目的 |
| 整合分散的信息片段 | 改变参与者的原意 |

## 🎓 为什么这是最重要的模式之一

### 1. 解决根本矛盾

```
AI 容易"幻觉"（凭空想象）
         ↕
简单提取太"机械"（缺少洞察）
         ↓
   场景重构的平衡点
   = 基于事实 + 深度理解
```

### 2. 适用范围最广

所有对话类内容都适用，是最通用的分析模式

### 3. 提升分析层次

```
Level 0: 机械复述 ❌
    ↓
Level 1: 简单提取 ⚠️
    ↓
Level 2: 场景重构 ✅ ← 我们在这里
```
