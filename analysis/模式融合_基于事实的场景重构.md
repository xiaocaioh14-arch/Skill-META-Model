## 一、问题诊断

### 当前两种极端

```
极端 1: 简单提取（过于机械）
├─ 逐字逐句引用原文
├─ 缺少场景感和上下文
└─ 像"法庭记录员"，不像"分析师"

极端 2: 主观臆断（不够客观）
├─ 凭空想象对话场景
├─ 添加原文中不存在的细节
└─ AI "幻觉"问题严重
```

### 用户的核心诉求

> **"我要的不是提取，而是重构"**

**重构的含义**:
- ✅ 基于事实（不胡编乱造）
- ✅ 还原场景（不仅是文字）
- ✅ 保留氛围（不机械摘录）
- ✅ 呈现动态（不是静态快照）

## 二、全新模式：基于事实的场景重构

### 模式定义

**模式 X: 基于事实的场景重构 (Evidence-Based Scene Reconstruction)**

**核心思想**:
> 深度理解对话内容后，基于事实还原整个对话场景、关系动态和情感氛围，而非机械地逐字提取。

**三大原则**:

1. **忠于事实** - 所有重构必须有原文依据，不添加臆想细节
2. **还原场景** - 重现对话的张力、转折、氛围
3. **保持客观** - 不做价值判断，只做场景描述

### 适用场景（扩展）

**所有对话类内容分析**:
- 📝 会议记录分析
- 💬 聊天记录分析
- 📞 客服对话分析
- 🎤 用户访谈分析
- 📧 邮件往来分析
- 🤝 商务谈判复盘

**关键特征**: 只要是"人与人的对话"，就适用

## 三、实现框架

### Level 1: 简单提取（❌ 不推荐）

```markdown
**决策过程**:
- 用户说："这个方案不行"
- 领导说："我觉得可以试试"
```

**问题**:
- 缺少上下文
- 看不出关系动态
- 无法理解决策逻辑

---

### Level 2: 场景重构（✅ 推荐）

```markdown
**决策场景重构**:

会议进入方案讨论环节。当项目经理提出A方案时，技术负责人
立即表达了顾虑，直言"这个方案在技术实现上有风险"。此时
出现了短暂的沉默。

领导者打破沉默，展现出开放的姿态："我觉得可以试试"。这句
话传递了两个信号：一是鼓励创新尝试，二是愿意承担风险。

技术负责人随后补充了具体的风险点，但没有继续坚持反对，显示
出对领导决策的尊重。最终团队达成共识，决定小范围试点。

> 事实依据：
> - "这个方案不行"（技术负责人原话）
> - "我觉得可以试试"（领导原话）
> - 会议记录显示此后无人继续反对
```

**优势**:
- ✅ 还原了对话的张力（技术顾虑 vs 领导鼓励）
- ✅ 呈现了关系动态（尊重权威 + 保留意见）
- ✅ 展示了决策过程（从分歧到共识）
- ✅ 基于事实（标注原文依据）

## 四、重构的四个维度

### 维度 1: 对话张力（Tension）

**识别对话中的关键转折点**:
- 分歧出现的时刻
- 观点碰撞的瞬间
- 立场转变的节点
- 共识达成的过程

**重构示例**:
```markdown
❌ 简单提取：
"A 说 X，B 说 Y"

✅ 场景重构：
"当 A 提出 X 观点时，B 立即提出了不同看法 Y。
这一分歧引发了 5 分钟的讨论..."
```

---

### 维度 2: 关系动态（Dynamics）

**捕捉对话中的权力关系和情感流动**:
- 谁主导了对话？
- 谁在回应谁？
- 谁的意见被采纳？
- 谁保持沉默？

**重构示例**:
```markdown
❌ 简单提取：
"老板说了三次，员工说了一次"

✅ 场景重构：
"会议中，领导连续提出了三个问题，主导了讨论方向。
员工们多数时候在倾听和记录，仅有一位资深员工
主动提出了补充建议。这显示出..."
```

---

### 维度 3: 情感氛围（Atmosphere）

**还原对话的情感基调**:
- 紧张 vs 轻松
- 严肃 vs 活跃
- 对抗 vs 协作
- 急迫 vs 从容

**重构示例**:
```markdown
❌ 简单提取：
"大家讨论了这个问题"

✅ 场景重构：
"讨论开始时，会议室气氛紧绷。当问题的严重性
被充分讨论后，团队成员开始更加坦诚地表达观点，
氛围逐渐转为协作和建设性..."
```

**事实依据**:
- 语言风格（正式 vs 口语化）
- 发言频率（频繁打断 vs 有序发言）
- 用词选择（批判性 vs 建设性）

---

### 维度 4: 决策逻辑（Logic）

**还原决策的演进路径**:
- 从问题发现到方案提出
- 从分歧讨论到共识达成
- 从犹豫不决到拍板决策

**重构示例**:
```markdown
❌ 简单提取：
"最后决定做 A"

✅ 场景重构：
"团队最初倾向于保守的 B 方案。但当技术负责人
指出 B 方案的隐性成本后，讨论方向发生转变。
经过成本收益的详细对比，团队最终选择了风险
稍高但长期收益更大的 A 方案。"
```

## 五、实现指令模板

### 核心指令

```markdown
## 场景重构指令

你的任务不是逐字逐句提取对话内容，而是**深度理解后重构场景**。

### 重构要求

1. **还原对话场景**
   - 谁在什么时刻说了什么？
   - 对话如何展开和演进？
   - 关键转折点在哪里？

2. **呈现关系动态**
   - 权力关系如何体现？（谁主导、谁跟随）
   - 情感流动如何变化？（紧张、放松、对抗、协作）
   - 立场如何演变？（坚持、妥协、转变）

3. **保持客观忠实**
   - 所有描述必须基于原文事实
   - 不添加原文中不存在的细节
   - 不做主观的价值判断

4. **标注事实依据**
   - 每个重构场景后，用引用块标注关键原文
   - 区分"明确说的"和"可推断的"

### 重构 vs 提取的区别

❌ **简单提取（不要这样）**:
"A: '这个方案不行'
B: '我觉得可以试试'"

✅ **场景重构（要这样）**:
"当项目经理提出A方案时，技术负责人表达了明确的顾虑，
认为技术实现有风险。领导者此时展现出开放的姿态，鼓励
团队尝试创新..."

> 事实依据：
> - 技术负责人："这个方案不行"
> - 领导："我觉得可以试试"
```

## 六、防止"过度重构"的边界

### 安全边界

| 可以做 ✅ | 不可以做 ❌ |
|---------|-----------|
| 还原对话的张力和转折 | 添加原文中不存在的情节 |
| 描述可观察的关系动态 | 臆测参与者的内心想法 |
| 基于用词推断情感氛围 | 编造参与者的动机和目的 |
| 整合分散的信息片段 | 改变参与者的原意 |
| 使用生动的场景化语言 | 使用夸张或修辞手法 |

### 判断标准

**"可追溯性测试"**:
> 重构的每一个描述，都应该能追溯到原文的某个细节

**示例**:

✅ **可以这样说**:
"技术负责人表达了明确的反对"
→ 追溯依据："这个方案不行"（原文）

❌ **不可以这样说**:
"技术负责人感到非常愤怒和沮丧"
→ 原文中没有情绪词，这是臆测

✅ **可以这样说**:
"领导展现出开放姿态，鼓励尝试"
→ 追溯依据："我觉得可以试试"（原文）+ 身份（领导）

## 八、实战案例对比

### 案例：会议记录分析

**原始文本**:
```
项目经理："我们可以用A方案"
技术负责人："A方案有风险"
领导："我觉得可以试试"
项目经理："那我们先做小范围测试"
```

---

**❌ Level 0: 机械复述**
```markdown
项目经理提出了A方案，技术负责人说有风险，
领导说可以试试，项目经理说先做测试。
```

**问题**: 像流水账，毫无洞察

---

**✅ Level 1: 简单提取（模式 12）**
```markdown
**方案讨论过程**:

项目经理提出了A方案。

> "我们可以用A方案"（项目经理）

技术负责人表达了顾虑。

> "A方案有风险"（技术负责人）

领导支持尝试。

> "我觉得可以试试"（领导）
```

**优势**: 有原文依据，客观
**问题**: 缺少场景感，看不出关系动态

---

**⭐ Level 2: 场景重构（模式 X）**
```markdown
**方案决策场景重构**:

会议进入方案选择环节。项目经理率先提出了A方案，
展现出主动推进的姿态。但这一提议立即遭到了技术
负责人的质疑，后者直接指出了技术实现的风险。

此时出现了关键的决策节点。领导者介入讨论，用一句
"我觉得可以试试"定下了基调——鼓励创新尝试。这个
表态传递了两个信号：一是对团队创新的支持，二是愿意
承担一定风险。

技术负责人在领导表态后没有继续坚持反对，显示出对
层级决策的尊重。项目经理顺势提出了折中方案——先做
小范围测试，既响应了技术风险的顾虑，又保留了尝试的
可能性。这一提议迅速获得了共识。

整个过程展现出：
- **决策风格**: 领导支持创新，但团队也会保留理性审慎
- **沟通模式**: 允许提出异议，但最终服从层级决策
- **风险态度**: 在控制风险的前提下，鼓励尝试新方案

> 事实依据：
> - "我们可以用A方案"（项目经理）
> - "A方案有风险"（技术负责人）
> - "我觉得可以试试"（领导）
> - "先做小范围测试"（项目经理）
> - 技术负责人在领导表态后未再发言（会议记录显示）
```

**优势**:
- ✅ 还原了对话的完整场景
- ✅ 展现了关系动态和决策逻辑
- ✅ 提取了组织文化信息
- ✅ 所有描述都可追溯到原文
