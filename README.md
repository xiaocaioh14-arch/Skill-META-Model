## 📦 核心组件说明

### 1. 🧬 SKILL_META_MODEL.md — Skill 开发元模型

**路径**: `core/SKILL_META_MODEL.md`

这是整个体系的**核心文档**，基于 8 个实战 Skill 提炼的开发框架。

**主要内容**:
- **11 种设计模式** — 从实战中总结的可复用模式
  - 模式 1: 静默执行协议
  - 模式 2: AI Fallback 降级机制
  - 模式 3: 双格式输出标配
  - 模式 4: 脚本分离架构
  - 模式 5: 阶段化工作流
  - 模式 6: 双报告架构
  - 模式 7: 多层估算模型
  - 模式 8: 加权评分系统
  - 模式 9: 模板驱动报告
  - 模式 10: 基于事实的场景重构 ⭐
  - 模式 11: 元认知与自我迭代 ⭐⭐

- **四阶段开发流程 (Eddie Method)**
  1. 启发式探索 → 2. 技能蓝图 → 3. 编写 SKILL.md → 4. 验证

- **SKILL.md 标准模板** — 开发新 Skill 的模板

**使用场景**: 
- 开发新 Skill 时参考设计模式
- 了解 Skill 开发最佳实践
- 统一团队的开发标准

### 2. 📋 AGENTS.md — 跨工具 AI 协作规则

**路径**: `core/AGENTS.md`

遵循 Linux Foundation / Agentic AI Foundation 推动的标准，让**多个 AI 工具**（Claude Code、Cowork、AntiGravity）遵循统一规则。

**主要内容**:
- **规则 1**: Skill 元模型自动更新
- **规则 2**: 会话交接（Handoff）
- **规则 3**: 禁止操作清单
- **规则 4**: 跨工具协作
- **规则 5**: 输出标准
- **规则 6**: 记忆与学习

**快速命令**:
| 命令 | 作用 |
|------|------|
| `/update-meta-model` | 更新 Skill 元模型 |
| `/handoff` | 保存会话上下文 |
| `/brief` | 生成当前状态简报 |
| `/remember [内容]` | 记录重要决策 |

**使用场景**:
- 在不同 AI 工具间保持上下文一致性
- 自动化元模型更新流程
- 记录和传递会话上下文

### 4. 🔧 meta-model-updater — 元模型更新器 Skill

**路径**: `meta-model-updater/`

这是一个**元工具 Skill**，具备「元认知」能力——能够分析其他 Skill 并自动更新元模型。

**核心能力**:
- 🧠 **模式识别** — 6 大维度分析框架
- 🔍 **相似度评估** — 智能去重逻辑
- ⚖️ **通用性评分** — 评估新模式价值
- 🔄 **自动更新** — 更新 SKILL_META_MODEL.md 和 SKILLS_INDEX.md

**触发方式**:
```bash
# 自然语言
"更新元模型"
"update meta model"

# 斜杠命令
/update-meta-model
/update-meta-model ~/Project/[skill-path]/
```

**解决的核心问题**:
> 将"需要记住的规则"转化为"可以调用的工具"

## 🚀 快速开始

### 开发新 Skill

1. **阅读元模型**: 查看 `core/SKILL_META_MODEL.md` 中的设计模式
2. **选择模式**: 根据需求选择合适的设计模式组合
3. **使用模板**: 按照 SKILL.md 标准模板创建你的 Skill
4. **更新索引**: 完成后说 "更新元模型" 来同步更新

### 跨工具协作

1. **初始化目录**: 
   ```bash
   mkdir -p ~/Project/.handoff
   mkdir -p ~/Project/.memory
   touch ~/Project/.memory/decisions.md
   ```

2. **使用命令**:
   - 说 `/update-meta-model` 更新元模型
   - 说 `/handoff` 保存会话上下文
   - 说 `/brief` 获取状态简报

## 🎯 设计哲学

```
┌─────────────────────────────────────────────────────────┐
│  🧙‍♂️ Skill 开发哲学                                       │
├─────────────────────────────────────────────────────────┤
│  1. 静默优先 — 不打断用户，一次性完整输出                    │
│  2. 双格式标配 — MD + HTML/PDF 始终成对出现                 │
│  3. 脚本分离 — 脏活累活交给 Python，Claude 专注智能处理      │
│  4. 优雅降级 — 正则 → AI Fallback → 人工兜底                │
│  5. 视觉极致 — 2026 前沿设计，毛玻璃/渐变/动效              │
│  6. 多视角输出 — 同一数据生成不同受众的专属报告             │
│  7. 数据质量追踪 — 每个维度标注来源和置信度                 │
│  8. 模板驱动 — 使用详细模板确保输出一致性                   │
│  9. 场景重构 — 基于事实还原场景，而非机械提取 ⭐            │
└─────────────────────────────────────────────────────────┘
```
